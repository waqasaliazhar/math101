§1

Injective (One-to-One) Functions

▼
Definition — Injective A function f : A → B is injective (or one-to-one) if no output comes from two different inputs.

Formally: if x ≠ y, then f(x) ≠ f(y).

Think of it this way: an injective function never "reuses" an output. Every output in the range has a unique input pointing to it.

✓ INJECTIVE
Each input maps to a different output. One element in B has no arrow pointing to it — that's okay for injectivity.
✗ NOT INJECTIVE
Two different inputs (2nd and 3rd) both map to the same output. That's a collision — injectivity fails.

The Horizontal Line Test

Horizontal Line Test If any horizontal line intersects the graph of f at more than one point, then f is not injective.

This is the graphical version of checking for "collisions." A horizontal line at height d represents the equation f(x) = d. If it crosses the graph twice, two different x-values give the same output.

Proving Injectivity Algebraically

Logic Trick — Contrapositive The statement "if x ≠ y then f(x) ≠ f(y)" is logically equivalent to its contrapositive:
if f(x) = f(y),  then  x = y
This contrapositive form is usually easier to prove: assume the outputs are equal, then show the inputs must be equal.
Example — Is f(x) = x² injective on ℝ? Attempt: Assume f(x) = f(y), i.e., x² = y².
Then x² − y² = 0, so (x − y)(x + y) = 0.
This gives x = y  or  x = −y.

Since we can't conclude x = y (we might have x = −y instead), f is not injective on ℝ.

Concrete counterexample: f(2) = 4 = f(−2), but 2 ≠ −2.
Example — What about f : [0, ∞) → ℝ, f(x) = x² ? Now assume f(x) = f(y) with x, y ≥ 0.
Again (x − y)(x + y) = 0.
Since x, y ≥ 0, we know x + y ≥ 0, so x + y = 0 only if x = y = 0.
In either case, x = y.   ✓ Injective!

The domain matters! Restricting to non-negative numbers kills the symmetry that caused collisions.
Example — f(x) = |x| is not injective The absolute value function maps both x and −x to the same output. For instance, |3| = 3 = |−3|.

Graphically, the V-shape fails the horizontal line test — any horizontal line above the vertex hits the graph twice.
§2

Surjective (Onto) Functions

▼
Definition — Surjective A function f : A → B is surjective (or onto) if every element in B is an actual output. That is:
B = Range(f)
In other words, every element of B has at least one input mapping to it — no element of B is "left out."
✗ NOT SURJECTIVE
The bottom element of B has no arrow pointing to it — it's never an output. So range ≠ B.
✓ SURJECTIVE
Every element in B has at least one arrow pointing to it. It's fine that the middle element gets two arrows.
Key Insight — The co-domain matters! Surjectivity depends on what you declare as the co-domain B. The same formula can be surjective or not depending on B.
Example — f(x) = x² with different co-domains f : ℝ → ℝ, f(x) = x² is NOT surjective: −1 ∈ ℝ has no input (since x² ≥ 0 always).

f : ℝ → [0, ∞), f(x) = x² IS surjective: for any y ≥ 0, we can find x = √y with f(x) = y.
Example — sin(x) and surjectivity f : ℝ → ℝ, f(x) = sin(x) is NOT surjective: the value 5 ∈ ℝ is never an output (since −1 ≤ sin(x) ≤ 1).

f : ℝ → [−1, 1], f(x) = sin(x) IS surjective: every value in [−1, 1] gets hit.

Counting Argument for Finite Sets

For finite sets A and B with #A and #B elements:

Size constraints If #A > #B, you can't have an injective map A → B (by the pigeonhole principle — too many inputs, not enough outputs).

If #A < #B, you can't have a surjective map A → B (not enough inputs to cover every output).

For a function to be both injective and surjective, we need #A = #B.
§3

Bijective Functions

▼
Definition — Bijective A function f : A → B is bijective if it is both injective and surjective.

Every element of B is hit (surjective), and hit exactly once (injective). This creates a perfect one-to-one pairing between A and B.
✓ BIJECTION
Each element on the left pairs with exactly one on the right, and vice versa. A perfect matching!

Bijective functions are also called invertible functions, because you can "reverse the arrows" to get a well-defined function going back.

§4

Inverse Functions

▼
Definition — Inverse Function If f : A → B is bijective, we can define f−1 : B → A by flipping inputs and outputs:
f−1(y) = x  ⟺  f(x) = y
The domains and ranges swap:   Df = Rf−1  and  Rf = Df−1

How to Find an Inverse

1 Write y = f(x).
2 Interchange x and y.
3 Solve for y in terms of x.
4 That solved y is your f−1(x).
Worked Example — f(x) = x/(x+1) Find f−1, and find Df, Rf, Df−1, Rf−1.
Domain of f:   Df = ℝ \ {−1}  (denominator ≠ 0).

Step 1:   y = x / (x + 1)
Step 2:   Swap: x = y / (y + 1)
Step 3:   Solve for y:
x(y + 1) = y
xy + x = y
y − xy = x
y(1 − x) = x
y = x / (1 − x)

Result:
f−1(x) = x / (1 − x)
Summary:
Df = ℝ \ {−1} = Rf−1
Rf = ℝ \ {1} = Df−1
Interesting Fact — Graphing Inverses The graph of f−1 is the reflection of the graph of f across the line y = x.

Why? If the point (a, b) is on the graph of f (meaning f(a) = b), then (b, a) is on the graph of f−1 (meaning f−1(b) = a). Swapping coordinates is exactly what reflecting over y = x does.

See how f(x) = x/(x+1) and its inverse reflect across y = x.

Exercise — Find the inverse Find f−1 for f(x) = 2x + 5.
y = 2x + 5 → swap: x = 2y + 5 → solve: y = (x − 5)/2.
f−1(x) = (x − 5) / 2
Exercise — Find the inverse Find f−1 for f(x) = (3x − 1) / (x + 2).
Swap and solve: x = (3y − 1)/(y + 2) → x(y + 2) = 3y − 1 → xy + 2x = 3y − 1 → y(x − 3) = −1 − 2x → y = (−2x − 1)/(x − 3).
f−1(x) = (−2x − 1) / (x − 3) = (2x + 1) / (3 − x)
§5

Infinity & Beyond — Cardinality

▼

Here's where bijections become surprisingly powerful: they let us compare the sizes of infinite sets.

Definition — Same Cardinality Two sets A and B have the same cardinality (same "size") if there exists a bijection f : A → B.

We write #A = #B.

ℕ and 𝕎 Have the Same Size

Let ℕ = {1, 2, 3, …} and 𝕎 = {0, 1, 2, …} (the whole numbers). Even though 𝕎 has an "extra" element (zero), they have the same cardinality!

Bijection ℕ → 𝕎 Define f(n) = n − 1. This is a bijection:
n ∈ ℕf(n) ∈ 𝕎
1→0
2→1
3→2
4→3
⋮⋮

ℕ and ℤ Have the Same Size!

This is more surprising: ℤ extends infinitely in both directions, yet it's "the same size" as ℕ.

Bijection ℕ (starting from 0) → ℤ The trick is to interleave: send even numbers to non-negatives, odd numbers to negatives.
f(n) = n/2  if n is even,   f(n) = −(n+1)/2  if n is odd
nf(n) ∈ ℤ
0→0
1→−1
2→1
3→−2
4→2
5→−3
6→3
⋮⋮
Like a zipper — weaving back and forth captures every integer!

The Hierarchy of Infinities

The cardinality of ℕ is denoted ℵ₀ ("aleph-naught"). Sets with this cardinality are called countably infinite.

Surprising fact The rationals ℚ also have cardinality ℵ₀ — same size as ℕ! (This can be shown using a clever diagonal argument.)

So ℕ, 𝕎, ℤ, and ℚ are all the same "size" of infinity.

But not all infinities are equal…

Cantor's Theorem Georg Cantor proved that the irrationals ℚc (and therefore ℝ) have a strictly larger cardinality than ℕ.
#ℚc = #ℝ > #ℕ
The cardinality of ℝ is denoted 𝔠 ("the continuum"). In fact, #(0,1) = #ℝ = 𝔠.
ℕ, ℤ, ℚ < ℝ, ℚc, (0,1) < 𝒫(ℝ) < ⋯

cardinality ℵ₀       cardinality 𝔠 = 2ℵ₀     and bigger…

Cantor's Power Set Theorem For any set A:   #A < #𝒫(A), where 𝒫(A) is the power set (set of all subsets) of A.

This means there's no biggest infinity! You can always build a bigger one:
ℵ₀ < 2ℵ₀ < 22ℵ₀ < 222ℵ₀ < ⋯
A never-ending chain of infinities!
The Continuum Hypothesis Is there an infinity "between" ℵ₀ and 𝔠?

This question, known as the Continuum Hypothesis, was the first of Hilbert's famous 23 problems (1900). In 1963, Paul Cohen proved that this question is independent of the standard axioms of mathematics — it can be neither proved nor disproved! One of the most remarkable results in all of mathematics.
Fun Exercises from the Lecture i) Can you find a bijective function from (0, 1) to (0, 2025)?
ii) Can you find a bijection from (0, 1) to [0, 1]?
i) Linear scaling works: f(x) = 2025x. This maps (0,1) bijectively onto (0, 2025).

ii) This is much trickier! A simple linear map won't work (it would map open endpoints to open endpoints). You need a clever construction — the standard approach uses a "Hilbert hotel" trick: handle a countable sequence of special points differently from the rest.

Practice Problems

Quiz — Classify the Function
Exercise 1 Prove algebraically that f(x) = 3x − 7 is injective on ℝ.
Assume f(x) = f(y):   3x − 7 = 3y − 7 → 3x = 3y → x = y. ✓
Exercise 2 Is f : ℝ → ℝ, f(x) = x³ injective? Surjective? Bijective?
Injective: If x³ = y³, then taking cube roots gives x = y. ✓
Surjective: For any y ∈ ℝ, x = ∛y gives f(x) = y. ✓
Bijective: Yes — it's both injective and surjective! ✓
Exercise 3 Find the inverse of f(x) = (2x + 1) / (3 − x). State the domains of f and f−1.
Swap: x = (2y + 1)/(3 − y) → x(3 − y) = 2y + 1 → 3x − xy = 2y + 1 → 3x − 1 = 2y + xy = y(2 + x) → y = (3x − 1)/(x + 2).
f−1(x) = (3x − 1) / (x + 2)
Df = ℝ \ {3},   Df−1 = ℝ \ {−2}.
Exercise 4 Give an example of a function ℝ → ℝ that is surjective but not injective.
f(x) = x³ − x works. It hits every real value (surjective) but, for example, f(0) = 0 = f(1) = f(−1), so it's not injective. Another classic example: f(x) = x · sin(x).
